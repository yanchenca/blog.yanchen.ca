Create RDD for HBase table with Kerberos in Spark on Yarn

Posted on 2016-12-07 Edited on 2022-01-25

In some cases, we need to read the whole HBase table into an RDD in Spark. However, if Kerberos authentication is enabled, using sc.newAPIHadoopRDD will cause authentication issue on the executor side. In this article, we describe a possible solution for this problem.

The way to work around this issue is to create your own RDD class, and override the getPartitions and compute methods to replace org.apache.spark.rdd.NewHadoopRDD used in sc.newAPIHadoopRDD. Please note that it’s easier to create a package named org.apache.spark.rdd in your own project, and put your RDD class in that package since it will use some classes private to spark.

In order to use this RDD class, you will need to follow the same procedure of newAPIHadoopRDD method from org.apache.spark.SparkContext class.
