Motorized LEGO Rally Car

Posted on 2018-06-15 Edited on 2022-01-25

In this post, we built a motorized LEGO rally car powered by Arduino with iPhone remote.

Video

Parts

  • LEGO Rally Car 42077
  • LEGO Power Functions L-Motor 88003 (DC Motor)
  • LEGO Power Functions Servo Motor 88004 (Servo Motor)
  • LEGO Power Functions Light 8870
  • 2 * LEGO Power Functions Extension Wire 8886: optional, only required if you don’t want to cut your motor/light wire.
  • Arduino Uno
  • L298N Dual H-bridge Stepper Motor Driver Module
  • ESP8266 ESP-01 WiFi Wireless Transceiver Module
  • a lot * Dupont Wire (Male to Male, Male to Female, Female to Female)
  • Breadboard: for testing
  • 3 * 18650 Battery
  • 18650 Battery Charger
  • 18650 Battery Case 3x
  • Soldering Iron, Solder, Rosin Soldering Flux: optional, only required if you want to solder the wires up.

Refer to Wire Connection sections for connecting these components.

The breadboard is used to connect dupont wires temporarily for testing. Refer to https://learn.sparkfun.com/tutorials/how-to-use-a-breadboard for how to use a breadboard.

Architecture

Arduino

Arduino is used to control the two motors used on the rally car.

Pinout

Power Supply

Incoming

There are four ways providing power to Arduino:

We will use two of them in this project.

  • USB: used to provide power when connected to computer, for uploading code, testing, etc.
  • 5V Pin: used to provide power from L298N in rally car.

Outgoing

The 3.3V pin is used to provide power to ESP8266 ESP-01 WiFi module in this project.

Digital Output Pins

Only pins marked as “~” are PWM pins, which are used to provide analog outputs. These pins are used to control the speed of DC motors and the angle of Servo motors.

Power Functions Motors/Light

  • DC motor refers a continuous rotation motor, providing speed to the rally car.
  • Servo motor refers to a motor which can only rotate within a certain angle range, providing direction to the rally car.

Pinout

Power Functions cables have the same pinout as follows.

DC Motor

Refer to L298N section for wire connection.

Servo Motor

Wire Connection

  • GND: Battery negative
  • 9V: Battery positive
  • C1: Arduino #5
  • C2: Arduino #9

Note that LEGO motors use 9V voltage normally. But it will work fine under 12V according to other people’s tests. The three 18650 batteries I used has a total of 10.8V as tested, although shown as 3.7V each in specs. Take your own risk here.

Control

C1 C2 State of Servo Motor
0 n% Rotate Clockwise (n% of 90°)
n% 0 Rotate Counterclockwise (n% of 90°)
0 0 Return to Zero Position

Light

Wire Connection

  • C1: Arduino #12
  • C2: Arduino #13

Control

C1 C2 State of Both Lights
1 0 On
0 1 On
0 0 Off

L298N

L298N is used to control DC motors. Each L298N is able to control 2 DC motors independently. We only need to control 1 DC motor in this project.

Pinout

Wire Connection

Battery

Provide 11.1V to L298N:

  • +12V: Battery positive
  • GND: Battery negative

Arduino

Provide 5V to Arduino:

  • keep 12V jumper (marked as JUMPER in pinout diagram) on
  • +5V: Arduino +5V
  • GND: Arduino GND

Connect input pins to Arduino:

  • remove ENABLE B jumper
  • ENABLE B (edge side of the two pins connected by the removed jumper): Arduino #3, providing PWM output to L298N
  • Input3: Arduino #7
  • Input4: Arduino #8

DC Motor

  • OUT3: DC Motor C1
  • OUT4: DC Motor C2

Control

ENABLE B (PWM) Input3 Input4 State of DC Motor B
0% X X Stop
n% 0 0 Brake
n% 0 1 Rotate Clockwise (speed n%)
n% 1 0 Rotate Counterlockwise (speed n%)
n% 1 1 Brake

ESP8266 ESP-01

Keep the AT firmware on the ESP8266 ESP-01 module. We are going to use AT commands to communicate between Arduino and ESP8266 through Serial.

Pinout

Wire Connection

  • GND: Arduino GND
  • VCC: Arduino 3.3V
  • CH_PD: Arduino 3.3V
  • U0RXD: Arduino #11
  • U0TXD: Arduino #10

Arduino

WiFiEsp library is used in this project. It will listen on port 10002 for incoming UDP packets.

Remember to put your desired ssid and pass in the code. You’ll need to connect to WiFi of this ssid with your iPhone later.

iOS

Alamofire and BlueSocket are used in this project.
